# Majestic RP Parser — C# / WPF

Порт парсера форума `forum.majestic-rp.ru` с Python (`parserv2.py`) на C# (.NET 8, WPF) с графическим интерфейсом.

## Запуск

```bash
cd MajesticParserCs
dotnet run
```

Требуется установленный Chrome. Драйвер `chromedriver` скачивается автоматически через встроенный Selenium Manager — отдельной настройки не нужно.

## Где хранятся данные

- Конфиг и кэш тредов: `%LOCALAPPDATA%\MajesticParser\` (`parser_config.json`, `thread_cache.json`)
- Результаты: папка, выбранная в интерфейсе → подпапки `majestic_laws_<дата_время>`

## Интерфейс

Главное — **дерево разделов и тем** (слева), повторяющее структуру форума:

- 📁 **Форум / подфорум** — раздел (папка). Нажми ▸, чтобы лениво подгрузить вложенные подфорумы и темы (браузер заходит на страницу при раскрытии).
- 📄 **Тема (закон)** — конечный материал. Галочка = парсить эту тему.
- ☑ **Галочка на 📁 разделе** = взять ВСЕ темы этого раздела и всех его подфорумов рекурсивно.
- 🚫 серая тема — пропала с форума, выбрать нельзя.

Кнопки над деревом:
- **➕ Добавить раздел** / **🗑 Удалить источник** — открывают всплывающее окно (overlay) прямо внутри приложения. Добавлять можно **только разделы (папки-форумы)** — ссылки `/forums/...`; ссылки на отдельные темы отклоняются с подсказкой.
- **🔄 Обновить** — перестроить дерево, **☐ Снять галочки**, **▾ Свернуть всё**.
- **Правый клик по узлу → 🗑 Удалить из дерева** — убрать ненужную папку или тему (пользовательский источник удаляется и из конфигурации; остальное — из вида до «Обновить»).

**Выбор сервера:** кнопка **«Загрузить серверы»** тянет разделы верхнего уровня с `https://forum.majestic-rp.ru/`. После выбора сервера в списке приложение само открывает его папку **«Организации»** (поиск по названию) и подгружает её содержимое в дерево.

**Тема:** интерфейс тёмный (единственная тема).

**Каскад галочек:** отметка на 📁 разделе автоматически отмечает всё внутри — и уже загруженные, и подгруженные позже подфорумы/темы.

Справа — настройки:
- **Режим обработки** — 7 режимов как в оригинале (текст / картинки / по post ID / по авторам).
- **Headless**, **папка результатов**, **возобновление** прерванного запуска (`majestic_laws_*`).
- **▶ Старт** собирает выбранные темы (раскрывая отмеченные разделы рекурсивно) и парсит. **⏹ Стоп** — мягкая остановка с возможностью продолжить.

## Соответствие Python → C#

| Python | C# |
|--------|-----|
| `requests` | `HttpClient` |
| `BeautifulSoup` | `AngleSharp` (CSS-селекторы) |
| `selenium` + `webdriver_manager` | `Selenium.WebDriver` (Selenium Manager сам качает драйвер) |
| `ThreadPoolExecutor` (картинки) | `Parallel.ForEachAsync` |
| `json` | `System.Text.Json` |
| консольное меню `input()` | WPF + MVVM |

### Что перенесено

- Анти-детект настройки браузера (CDP-скрипт, user-agent, excludeSwitches)
- `safe_driver_get` с ретраями, `wait_page_loaded` (ожидание readyState + проверка на challenge)
- Синхронизация кэша тредов (new / updated / missing), сортировка по ID
- Рекурсивный обход подфорумов с защитой от циклов (`visited`)
- Извлечение текста: раскрытие спойлеров, удаление служебных блоков, `get_text(separator)`
- Извлечение картинок: приоритет `bbImageWrapper`, fallback по `<img>`, фильтры доменов/классов/путей
- Параллельное скачивание картинок + **дедупликация уже скачанных** (resume)
- Имена файлов: sanitize, smart-truncate, stop-words, ID-суффикс
- Per-thread `.txt`, общий `ALL_MAJESTIC_LAWS.txt` (собирается с диска), `manifest.json`
- **Возобновление прерванного запуска** через `progress.json` + маркер `_DONE`
- Лимиты страниц (`MaxForumPages` / `MaxThreadPages`) от зацикливания

### Отличие от консольной версии

Интерактивный пошаговый выбор подфорумов (меню «выбрать подфорумы вручную по номерам») заменён на полноценное **дерево** (`TreeView`) с ленивой подгрузкой: разделы можно раскрывать на любую глубину, отмечать отдельные темы или целые разделы галочкой.

## Структура

```
Models/Models.cs          — DTO (конфиг, треды, картинки, настройки)
Services/
  AppConstants.cs         — все константы из parserv2.py
  ConfigService.cs        — конфиг + кэш тредов + источники
  UrlHelper.cs            — нормализация URL, ID, фильтры картинок
  FilenameHelper.cs       — имена файлов
  BrowserService.cs       — Selenium-драйвер, навигация, ожидание
  HtmlHelper.cs           — AngleSharp-утилиты (замена BeautifulSoup)
  MessageParser.cs        — текст сообщения + post id
  ImageService.cs         — кандидаты, параллельное скачивание, manifest
  ForumScraper.cs         — список тредов, подфорумы, синхронизация кэша
  ThreadParser.cs         — страницы темы, сборка результата
  ParserEngine.cs         — жизненный цикл браузера + прогон с resume
ViewModels/               — MVVM (MainViewModel, команды, элементы)
MainWindow.xaml(.cs)      — интерфейс
```
